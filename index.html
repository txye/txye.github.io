5. Longest Palindromic Substring

Given a string s, find the longest palindromic substring in s. You may assume that the maximum length of s is 1000.

  • 思路：L[i][j] = true，表示字符串第i个位置到第j个位置为回文子串。初始化所有L[i][i]为true。且有L[i][j] = true if (s[i] == s[j] && (j - i) == 1), L[i][j] = L[i + 1][j - 1] if (s[i] == s[j])

15. 3Sum

Given an array S of n integers, are there elements a, b, c in S such that a + b + c = 0? Find all unique triplets in the array which gives the sum of zero.

思路：首先对列表升序排序。固定第一个数，然后设置两个移动指针，一个指针从第一个数后面开始，另一个指针从尾部开始。 需要注意的是，中间要跳过重复的数。

  • 思路：和3Sum一样，只是需要先固定前两个数

排序

冒泡排序 (稳定)

  • 两两比较交换
  • 平均时间复杂度 O(n^2),最好情况O(n)
  • 代码

插入排序（稳定）

  • 每次将一个待排序的记录，按其关键字大小插入到前面已经排好序的子序列中的适当位置，直到全部记录插入完成为止
  • 平均时间复杂度O(n^2)，最好情况O(n)
  • 代码

合并排序（稳定）

  • 将两个或两个以上的有序数表合并成一个有序表
  • 平均时间复杂度 O(nlogn), 最好情况最坏情况均为 O(nlogn)
  • 可用于找逆序对

选择排序（不稳定）

  • 首先在未排序序列中找到最小元素，存放到排序序列的起始位置，然后，再从剩余未排序元素中继续寻找最小元素，然后放到排序序列末尾。以此类推，直到所有元素均排序完毕。具体做法是：选择最小的元素与未排序部分的首部交换，使得序列的前面为有序。
  • 最好,最坏,平均时间复杂度O(n^2)

堆排序（不稳定）

  • 利用完全二叉树中双亲节点和孩子节点之间的内在关系，在当前无序区中选择关键字最大(或者最小)的记录。也就是说，以最小堆为例，根节点为最小元素，较大的节点偏向于分布在堆底附近。
  • 时间复杂度O(nlogn)

快速排序 (不稳定)

  • 找到一个标杆，左边比标杆小，右边比标杆大。然后对标杆左边和右边进行同样的操作。
  • 平均时间复杂度 O(nlog(n))，最坏情况下即排好序的情况，时间复杂度为O(n^2)
  • 可用于找第k大的数，找中位数
  • 代码

T1

在一个二维数组中，每一行都按照从左到右递增的顺序排序，每一列都按照从上到下递增的顺序排序。请完成一个函数，输入这样的一个二维数组和一个整数，判断数组中是否含有该整数。

  • 解决思路：

  • 时间复杂度：O(M+N）, M为行数，N为列数

T4

根据 前序遍历和中序遍历重建二叉树
输入某二叉树的前序遍历和中序遍历的结果，请重建出该二叉树。假设输入的前序遍历和中序遍历的结果中都不含重复的数字。例如输入前序遍历序列{1,2,4,7,3,5,6,8}和中序遍历序列{4,7,2,1,5,3,8,6}，则重建二叉树并返回。

  • 解决思路：
    根据前序遍历确定根的元素，在中序遍历中找到根的位置，然后可以分别得到左子树和右子树的前序遍历和中序遍历，再递归地建立就好

  • 时间复杂度：O(n*log(n)）

  • 空间复杂度：O(n)

  • 代码
